# 貢献ガイドライン (Contributing Guidelines)

プロジェクト名：**chosei-kun** (Next.js + Cloudflare D1)

このプロジェクトへのご関心、誠にありがとうございます！オープンソースコミュニティへの貢献は、バグ報告から新機能の提案、コードの修正まで、どのような形であれ大歓迎です。

## 貢献の方法

### 1. バグ報告

バグを発見した場合は、GitHubのIssuesから報告してください。

- 報告前に、既に同じIssueが存在しないか確認してください。
- バグ報告の際は、再現手順、期待される動作、実際の動作、実行環境について詳しく記載してください。
- 可能であればスクリーンショットも添付してください。

### 2. 機能提案

新機能のアイデアがある場合は、Issuesで提案してください。

- 背景、目的、考えられる実装アプローチについて記載してください。

### 3. コードの修正・機能追加（Pull Requests）

PRの送信は以下の手順でお願いします。

1. **リポジトリのフォーク**
   プロジェクトをフォークし、手元にクローンしてください。
2. **ブランチの作成**
   `main` ブランチから作業用ブランチを切ってください。
   （例：`git checkout -b feature/your-feature-name`）
3. **開発環境の構築**
   `README.md` を参考に、手元で動作する環境を用意してください。
4. **変更の実装**
   - コードの規約に従ってください（既存のコードスタイルに合わせる、等）。
   - 必要に応じて `npm run lint` などで静的解析を通してください。
5. **コミット**
   意味のあるコミットメッセージを記述してください。
6. **Push と Pull Request の作成**
   変更をリポジトリへ Push した後、本リポジトリに対して Pull Request を作成します。
   PRの概要には、変更内容とその理由を記載してください。

## 開発環境のセットアップ

開発環境のセットアップについては、`README.md`の「Getting Started」セクションをご確認ください。
